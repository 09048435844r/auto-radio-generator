# UI構造 抜本的整理・改善プラン

## 📋 現状の問題点

### 1. タブ構造の問題
- **Generator タブが重複**：同じ名前のタブが2つ表示されている
- **Manual タブと Generator タブの機能が重複**：
  - Generator タブ内に「通常モード/こだわりステップモード」の切り替えがある
  - Manual タブにも同様の「こだわりステップモード」がある
  - ユーザーが混乱する

### 2. UI要素の問題
- **アコーディオンが展開できない**：「🛠 こだわりステップモード」が押せない状態
- **モード選択が二重**：ラジオボタンとタブで同じ機能を提供している

### 3. ユーザビリティの問題
- **導線が不明確**：どのタブで何ができるのか分かりにくい
- **機能の重複**：同じ機能が複数の場所に存在

### 4. 機能の欠落
- **リサーチのみモードが無い**：リサーチデータだけを作成して保存する機能が存在しない
  - 現状は「台本のみ作成」ボタンはあるが、リサーチデータを保存する機能がない
  - リサーチ結果を再利用したい場合に不便

---

## 🎯 整理改善の方針

### 基本方針
1. **タブの役割を明確化**：各タブで何ができるかを明確にする
2. **重複を排除**：同じ機能は1箇所にまとめる
3. **シンプル化**：ユーザーが迷わないUI構造にする

### Phase 2: モード切り替えUIの統一（3-4時間）

#### 目的
- 全自動モードとステップモードをラジオボタンで切り替え
- UI の重複を解消

#### 実装手順

1. **ラジオボタンの追加**
   ```python
   # create_generator_tab() 内の最上部に追加
   mode_selector = gr.Radio(
       choices=["🚀 全自動モード", "🛠️ ステップモード"],
       value="🚀 全自動モード",
       label="制作モード",
       info="モードを切り替えて使い分けられます"
   )
   ```

2. **モード切り替え関数の実装**
   ```python
   def toggle_mode(mode: str):
       """モード選択に応じてUIを切り替え"""
       if mode == "🚀 全自動モード":
           return (
               gr.update(visible=True),   # 全自動モードUI
               gr.update(visible=False)    # ステップモードUI
           )
       else:
           return (
               gr.update(visible=False),   # 全自動モードUI
               gr.update(visible=True)     # ステップモードUI
           )
   ```

3. **UIコンポーネントの再配置**
   - 全自動モードのUIを `gr.Column(visible=True)` でラップ
   - ステップモードのUIを `gr.Column(visible=False)` でラップ
   - `mode_selector.change()` イベントで切り替え

4. **イベントハンドラーの更新**
   ```python
   mode_selector.change(
       fn=toggle_mode,
       inputs=[mode_selector],
       outputs=[auto_mode_column, step_mode_column]
   )
   ```

#### 影響範囲
- `create_generator_tab()` 関数
- 既存のUIコンポーネント配置

#### リスク
- 既存のイベントハンドラーとの競合

## 🔍 リサーチのみモードの使い方

### 保存されるファイル
```
data/research/research_20260320_143025_AI倫理について.jsonl
```

### ファイル形式
```json
{
  "timestamp": "2026-03-20T14:30:25",
  "theme": "AI倫理について",
  "mode": "トリビア (雑学)",
  "angle": "AI倫理の多角的視点",
  "queries": ["AIの倫理的課題", "AI規制の動向", "..."],
  "content": "リサーチ結果のテキスト...",
  "sources": ["https://...", "..."]
}
```

### 手動モードでの利用方法
1. リサーチのみモードで保存したファイルを開く
2. `content` フィールドの内容をコピー
3. 手動制作タブの「Step A: リサーチ結果から台本生成」に貼り付け
4. 台本を生成
